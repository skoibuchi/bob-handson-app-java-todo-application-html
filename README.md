## Bobのモード

Bobにはそれぞれに特化したモードが存在します。

### Planモード
- 実装前の計画・設計・戦略立案に特化
- 技術仕様書、アーキテクチャ設計、要件定義書の作成に使用

### Agentモード
- コードの作成・修正・リファクタリングから、コマンド実行まで幅広く対応
- 新機能実装、バグ修正、テストコード作成、単体テスト実行に使用

### Askモード
- 説明、ドキュメント、技術的な質問への回答に特化
- コード変更は行わず、分析と情報提供のみ
- Mermaid図を使った視覚的な説明が可能

### Java Modernizationモード
- Javaコードが含まれる場合に表示されます
- Javaバージョンのアップグレードとアプリケーションの再プラットフォーム化に特化
- WebSphereからLibertyへの移行をサポート
- レガシーJavaアプリケーションのモダナイゼーションに使用

### モード間の遷移フロー

```mermaid
graph LR
    A[Plan モード<br/>計画・設計] -->|実装開始| B[Agent モード<br/>実装・テスト]
    A -->|質問・確認| C[Ask モード<br/>質問・理解]
    B -->|質問・確認| C
    C -->|設計が必要| A
```

**モード遷移の説明：**
- **Plan → Agent**: 設計完了後、実装フェーズへ移行
- **Agent → Ask**: 実装中に質問や確認が必要な場合
- **Ask → Plan**: 質問の結果、再設計が必要と判断された場合

## 開発ルールの設定方法

### 1. Bobウィンドウ上部の歯車マーク→「モード」から設定
各モード共通のカスタム指示が可能。

1. モードにて、編集するモードを指定
2. 編集ボタンを押して、編集画面を開く
3. 「モード固有のカスタム指示」にて、各モードにてAIに共通で指示する内容を記載
4. 保存ボタンを押して、設定を保存

## ハンズオンの流れ

以下の流れでハンズオンを実施します。

### 1. 既存アプリの解析
- 既存アプリの設計書を作成
- テンプレートに沿った項目で設計書が生成されていることを確認

### 2. 追加機能の要件定義
- Bobと要件定義
- Bobとやりとりを行い、仕様を詰める
- 仕様に沿った設計書が生成されることを確認

### 3. 実装
- Bobと実装
- 設計書に沿ったコードが生成されることを確認

### 4. 単体テスト
- 要件に沿ったテストコードが生成されることを確認
- テストコードを実行し、パスすることを確認

## 1. 既存アプリの解析

Planモードにて、既存アプリをBobに解析させて、設計書を作成させます。  
設計書の内容はテンプレートに沿った内容が出力されるようにします。

### 手順

1. **Planモードに設定**

2. **Bobに既存アプリを解析し、設計書を作成するようにリクエストを出します**
   ```
   タスク管理Webアプリケーションを解析して、設計書を作成してください。
   ```

3. **Bobがプロジェクト内にあるファイルから情報を読み取ろうとするので、適宜承認していきます**

4. **不足している情報や、設計書の出力方法について聞かれた場合は、適宜答えていきます**

5. **情報が揃ったと判断すると、BobがTodoリストを作成し、計画を立てるので、問題がなければ承認します**  
   工程が不足しているなど問題があればBobに伝えます。

6. **適宜設計書が作成されるので、内容が問題なければ承認、問題があれば適宜Bobに伝えて修正を促します**
   1章ごとに出力することを推奨しています。もし、Bobがファイルを一括で出力しようとした際は、以下のようにリクエストを送信してください。
   ```
   1章ごとに出力してください。
   ```

## 2. 追加機能の要件定義

既存アプリに追加機能の要件定義をリクエストします。

### 手順

1. **Planモードに設定**

2. **Bobに追加機能設計書を作成するようにリクエストを出します**
   ```
   期限が3日以内のタスクを通知する機能を追加したい。
   ```
   
   ※Bobから仕様を詰めるための質問が来る場合は適宜答えていきます。
   
   例：
   ```
   ダッシュボード一覧に「期限が近いタスク一覧」セクションを追加して一覧表示する
   ```

3. **Bobがプロジェクト内にあるファイルから情報を読み取ろうとするので、適宜承認していきます**

4. **不足している情報や、設計書の出力方法について聞かれた場合は、適宜答えていきます**

5. **情報が揃ったと判断すると、BobがTodoリストを作成し、計画を立てるので、問題がなければ承認します**  
   工程が不足しているなど問題があればBobに伝えます。

6. **適宜設計書が作成されるので、内容が問題なければ承認、問題があれば適宜Bobに伝えて修正を促します**  
   設計書の作成が行われずに実装を開始しそうな時は、適宜Bobに設計書を作成するように指示してください。  
   設計書は既存設計書を更新するようにしてください。  

   例:
   ```
   設計内容を既存の設計書を更新する形で出力してください。
   ```

## 3. 実装

作成した設計書を基に、Bobに実装をリクエストします。

### 手順

1. **Bobに実装をリクエストします**
   
   - `2. 追加機能の要件定義`のチャットの続きで、以下のリクエストを送信します
      ```
      実装を進めてください  
      ```
   - もし、チャットを削除・新規プロジェクトとして作成した場合は、新しいチャットで以下のリクエストを送信します。
      ```
      設計書を基に実装を行なってください
      ```

2. **Agentモードでない場合は、BobがAgentモードへの変更を要求してくるので、承認します**

3. **必要に応じて、Bobが既存ファイルを確認するため、承認します**

4. **実装が開始したら、適宜内容を確認し、承認・保存していきます**
   
   ※Bobがコマンドを実行しようとする場合があります。問題ないか確認してください。

5. **実装が完了したら、動作確認を行います**  
   
   - HTMLファイルを直接開くコマンドをBobに実行させる、もしくはhtml-task-manager-app/index.html`をユーザーで直接ブラウザで開いてください。  
      ※Bobから以下のようにローカルサーバーを起動するコマンドを提示される場合がありますが、拒否してください。

     ```sh
     # 拒否するコマンド
     cd html-task-manager-app && python -m http.server
     ```
     
     以下のようなコマンドであれば、ブラウザで開く動作となります。承認してください。

     ```sh
     # 承認するコマンド
     open html-task-manager-app/index.html
     ```
   
   - 問題がなければ実装を終了します。以下のように追加機能が実装された状態になれば成功です。

   <img src="./images/sample-java-app-enhanced.png" alt="サンプルアプリ追加機能" width="800">

## 4. 単体テスト

Bobに作成したアプリケーションの単体テスト工程の実施をリクエストします。

### 手順

1. **Planモードに設定されていることを確認します**

   Planモードからだとテスト実施計画書の作成から実施してくれます。

2. **Bobに単体テストの実施をリクエストします**
   ```
   タスク管理Webアプリケーションの単体テストの実施をお願いします。
   ```
   
   実施計画書が作成されて、計画書が出力されない場合は以下のようにリクエストしてください：
   ```
   カバレッジ率を含めた実施計画書を出力してください。
   ```

3. **内容に問題がなければテストの実装をリクエストします**
   ```
   テストを実装してください。
   ```

4. **Bobにテストコード実行を実施させ、最後にテストレポートを出力させます**
   ```
   テストを実行し、レポートを出力してください。
   ```
   
   ※ テストはブラウザを使用するため、Agentモードで実行されることを確認してください。

## まとめ

このハンズオンでは、Bobを使用して以下の開発プロセスを体験しました：

1. 既存アプリケーションの解析と設計書作成
2. 新機能の要件定義と設計
3. 設計書に基づいた実装
4. 単体テストの計画と実施

Bobの各モードを適切に使い分けることで、効率的な開発が可能になります。

## トラブルシューティング（2026/6/12時点）
- BobがWebサーバー起動などで、応答が返ってこないかつ動作し続けるコマンドを実行するケースがあります。
   ```
   # 例
   python -m http.server
   ```
   推奨される手段としては、本アプリはブラウザで直接htmlを開くだけで起動できるため、Webサーバーの起動は行わず、htmlを直接開くような手順を実施、もしくはBobに指示してください。

- BobへのAPIリクエストが失敗し続ける場合があります。
   - ハンズオンでの回避策としては、新しいチャットを立ち上げて、Bobに続きを実行するよう指示してください。
   - もしくは、再試行をONにしておくと、エラー回避の挙動をしてくれることを確認しています。

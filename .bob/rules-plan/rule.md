# 設計書生成ルール
1. テンプレート
    - (**重要**) 設計書生成の際は以下のテンプレートを参照し、準拠した項目を作成すること。
        - 要件定義書: templates/requirements_template.md
        - 基本設計書: templates/basic_design_template.md
        - 詳細設計書: templates/detail_design_template.md
    - テンプレートにない項目が、設計する上で必要な場合は、適宜追加すること。必要に応じて、新規ファイルを作成すること。

2. 作成順序
    - 設計書は以下の順で作成することを推奨する。
        1. 要件定義書
        2. 基本設計書
        3. 詳細設計書

3. 作成ルール
    - 共通: 
        - (**重要**) 1つのファイルにつき、一度出力せずに、必ず1章ごとに生成して保存すること。これはBobの処理とメモリの制約で、一度に大量の文章を生成するとエラーで停止してしまうのを回避するためである。
        - 設計変更が発生した場合は、全ての設計書について、影響を調査し、影響を受けた設計書を修正すること。
    - 要件定義書:
        - 詳細設計は他文書を参照させても良い。（例：画面設計に関しては、detail_design.mdを参照。）
    - 基本設計書：
        - 設計書には詳細なコードまでは記述しないこと。
    - 詳細設計書:
        - 設計書には詳細なコードまでは記述しないこと。
        - 必要であればコード例を含むことは問題ない。
